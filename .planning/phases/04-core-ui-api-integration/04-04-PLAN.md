# Plan: Fix gRPC-Gateway Registration (HTTP 500 Error)

**Type:** Gap Closure  
**Status:** ready

## Problem

API endpoint `/tournament/v1/public/namespace/{namespace}/tournaments` returns HTTP 500 Internal Server Error, blocking all tournament viewing functionality.

**Root Cause:** gRPC-Gateway using `RegisterTournamentServiceHandlerFromEndpoint` (network-based client connection) causing HTTP/2 PROTOCOL_ERROR. Should use `RegisterTournamentServiceHandlerServer` for direct in-process server invocation.

**Source:** 04-core-ui-api-integration-UAT.md Test #2 (blocker severity)

## Objective

Fix gRPC-Gateway initialization to use direct in-process server registration instead of network-based connection, enabling tournament API endpoints to return data successfully.

## Success Criteria

1. Tournament list API endpoint returns HTTP 200 with valid JSON
2. Tournament detail API endpoint returns HTTP 200 with valid JSON  
3. Participant list API endpoint returns HTTP 200 with valid JSON
4. Browser displays tournament cards with name, description, status, participant count
5. No HTTP 500 errors in browser console or server logs

## Changes Required

### 1. Update Gateway Initialization Function

**File:** `pkg/common/gateway.go`

Add new function `NewGatewayWithServer` that uses `RegisterTournamentServiceHandlerServer`:

```go
// NewGatewayWithServer creates a gateway mux with direct server registration
func NewGatewayWithServer(ctx context.Context, tournamentServer tournamentpb.TournamentServiceServer, basePath string) (*runtime.ServeMux, error) {
    // Create gateway mux with same options as NewGateway
    gwmux := runtime.NewServeMux(
        runtime.WithMarshalerOption(runtime.MIMEWildcard, &runtime.JSONPb{
            MarshalOptions: protojson.MarshalOptions{
                UseProtoNames:   true,
                EmitUnpopulated: true,
            },
            UnmarshalOptions: protojson.UnmarshalOptions{
                DiscardUnknown: true,
            },
        }),
        runtime.WithIncomingHeaderMatcher(CustomMatcher),
        runtime.WithErrorHandler(CustomErrorHandler),
    )

    // Register tournament service using direct server invocation (not network connection)
    if err := tournamentpb.RegisterTournamentServiceHandlerServer(ctx, gwmux, tournamentServer); err != nil {
        return nil, fmt.Errorf("failed to register tournament service handler: %w", err)
    }

    return gwmux, nil
}

// CustomErrorHandler provides better error messages for debugging
func CustomErrorHandler(ctx context.Context, mux *runtime.ServeMux, marshaler runtime.Marshaler, w http.ResponseWriter, r *http.Request, err error) {
    // Log the error for debugging
    log.Printf("Gateway error: %v (path: %s)", err, r.URL.Path)
    
    // Use default error handler
    runtime.DefaultHTTPErrorHandler(ctx, mux, marshaler, w, r, err)
}
```
